Running Z-Wave-Protocol-Controller on Raspberry Pi

Note: For reference documentation please refer to related chapter of UnifySDK documentation

The following chapters, may be also relevent for z-wave-protocol-controller users/developers.

The Z-Wave-Protocol-Controller targets our reference platform, a Raspberry Pi 4 (RPi4). It is therefore recommended to set up a RPi4 for evaluating the full set of functionalities.

For details on setting up a bare RPi4, follow the setup instructions in the Official Raspberry Pi User Guide, and ensure a supported radio module is attached to your RPi4.

For set up and running quickly to test out Z-Wave-Protocol-Controller on the Raspberry Pi refer Setting up Raspberry Pi for Z-Wave-Protocol-Controller

The rest of this guide focuses on how to set up all of the applications and services on a RPi4.

For running Z-Wave-Protocol-Controller Framework it is required to have a running MQTT broker to which the Z-Wave-Protocol-Controller Framework can connect. To get an MQTT broker installed on the RPi4:

pi@raspberrypi:~ $ sudo apt update
pi@raspberrypi:~ $ sudo apt install mosquitto

For more information regarding logging and configuration of eg. the Mosquitto broker, see the Z-Wave-Protocol-Controller Host SDK User Guide.

How to install packages

The Z-Wave-Protocol-Controller Host SDK distribution targets the reference platform Debian Bookworm platform. Binary Debian packages are provided with the SDK and they are available in a Zip file eg. unify_x.y.z_arm64.zip on Z-Wave-Protocol-Controller’s GitHub release page.

The simplest way to install the Z-Wave-Protocol-Controller components is to use the apt tool. Transfer the .deb packages to the target via eg. SCP. Make sure to use either a local relative path with ./ or absolute paths. Otherwise apt will look for the package in the wrong place and fail.

Install a specific Z-Wave-Protocol-Controller package:

pi@raspberrypi:~ $ sudo apt install uic-sdk-dir/uic-<component>_<version>_arm64.deb

Install all packages:

pi@raspberrypi:~ $ sudo apt install uic-sdk-dir/*.deb

Update installed packages:

pi@raspberrypi:~ $ sudo apt reinstall uic-sdk-dir/*.deb

Verify the configuration

After installing the desired Z-Wave-Protocol-Controller Host SDK packages, make sure the configuration is set correctly.

pi@raspberrypi:~ $ cat /etc/uic/uic.cfg
zpc:
  serial: /dev/ttyUSB0
  rf_region: EU
  datastore_file: /var/lib/uic/zpc.db

If your configuration is wrong, you can either edit the file directly or reconfigure a specific package. After making changes to the configuration make sure to either reboot or restart the service.

pi@raspberrypi:~ $ sudo dpkg-reconfigure uic-zpc

Verifying if an application is running

The Z-Wave-Protocol-Controller applications all install SystemD service files for handling the starting and stopping of the applications on a RPi4. You can check if a Z-Wave-Protocol-Controller Host SDK application is running by using systemctl. For information on how to handle SystemD services using systemctl, see the systemctl man pages.

Here we can see that the uic-zpc service is loaded and active:

pi@raspberrypi:~ $ systemctl status uic-zpc.service
