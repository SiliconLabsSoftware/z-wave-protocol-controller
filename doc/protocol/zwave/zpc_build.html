ZPC Build Guide

How to build ZPC

ZPC is supporting native building on regular Linux system, Debian 12 amd64 is the reference OS so it should work without effort on derivates systems (eg: Raspbian12 aarch64, Ubuntu, WSL2 etc).

Extra adaptation might be needed to support other Linux distributions, (eg: RPM, Yocto, OpenWRT etc).

Native Build on target device

The project is using cmake build system, so standard directives can be used:

mkdir build
cmake ..
cmake --build .

But if you are using a supported OS (debian currently) we suggest first to setup your system and use the helper script to assist you to setup developer system, and build everything from sources to packages.

sudo apt install make
./helper.mk help
./helper.mk setup
./helper.mk

This script is straightforward to read and can be adjusted for others setup.

Build for foreign architecture

To speed up the build process, native ARM build can be deported to the host using different containerization techniques (systemd, chroot, qemu, binfmt). Try the build-rootfs.sh script for more information.

Docker build

A reference Dockerfile is also used for CI, it can be used also for developers:

This one liner will setup the build environment (using ./helper.mk setup) and pull dependencies (UnifySDK…) and build applications and related packages (deb files)

docker build https://github.com/SiliconLabsSoftware/z-wave-protocol-controller.git#main
